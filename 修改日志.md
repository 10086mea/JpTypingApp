# 修改与问题日志

## 2026-03-20

### 问题 1: 引入 libcurl 和 nlohmann_json 后 CMake 配置失败

*   **现象**: 首次在 `CMakeLists.txt` 中添加 `FetchContent` 来拉取 `libcurl` 和 `nlohmann_json` 后，执行 CMake 配置失败。
*   **初步错误信息**: `命令语法不正确。`
*   **详细错误信息**: `Make Error at build/_deps/json-src/CMakeLists.txt:1 (cmake_minimum_required): Compatibility with CMake < 3.5 has been removed from CMake.`

### 第一次修复尝试

*   **分析**:
    1.  根据你的经验，这可能是 `FetchContent` 在处理子项目时的一个策略（Policy）问题。
    2.  `CoroTask.h` 缺少 `Task<void>` 的特化，会导致后续编译失败。
    3.  `libcurl` 的链接目标名在 `FetchContent` 中应该是 `curl::libcurl` 而不是 `libcurl`。
*   **修改记录**:
    1.  **`CMakeLists.txt`**:
        *   在 `include(FetchContent)` 后添加 `set(CMAKE_POLICY_DEFAULT_CMP0077 NEW)`。
        *   将 `target_link_libraries` 中的 `libcurl` 修改为 `curl::libcurl`。
    2.  **`include/CoroTask.h`**:
        *   在文件末尾添加了 `Task<void>` 的完整特化实现。
*   **结果**: **失败**。同样的 `cmake_minimum_required` 错误依然存在。
